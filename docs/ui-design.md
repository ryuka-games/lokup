# レポート設計

## 出力形式

CLI → HTML レポート（単一ファイル）。ブラウザで開いて閲覧する。

```
lokup facebook/react --output report.html
→ report.html を生成
→ ブラウザで開く
```

## 3段階開示（Progressive Disclosure）

経営者は上だけ見て、技術者は下を見る。

### 第1層: 総合グレード
- A〜D の1文字評価
- 色分け（緑→赤）で一目で状態が分かる
- 経営者はここだけ見れば十分

### 第2層: カテゴリカード
- Velocity / Quality / TechDebt / Health の4カテゴリ
- 各100点満点のスコア + グレード + 診断コメント
- Chart.js のバーチャートで比較表示

### 第3層: 詳細セクション
- リスク一覧（重要度別: 🔴 High / 🟡 Medium / 🟢 Low）
- DORAメトリクス（デプロイ頻度、変更失敗率、MTTR）
- 投資比率（Feature / BugFix / Refactor）
- 巨大ファイル・古い依存の一覧
- 時間帯別・曜日別コミット分布
